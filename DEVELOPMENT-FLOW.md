# Tirisfal 开发流程

## 代码质量流程

### 1. 编码阶段
- 编写代码
- 本地测试（wrangler dev）
- 确认功能正常

### 2. 审查阶段
- 代码自审
- 检查安全性
- 检查性能

### 3. 提交阶段
- 运行构建（npm run build）
- 运行测试
- 创建有意义的 commit message
- 推送到远程

### 4. 合并前
- 创建 PR
- 等待 review
- 解决 review 意见
- 合并到主分支
